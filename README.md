# Поисковая система

Поисковая система для быстрого поиска и обработки новостных публикаций "РИА Новости" с возможностью получения обобщённого ответа на вопрос (RAG).

## Описание

Приложение представляет собой систему поиска, которая позволяет пользователям быстро и эффективно искать релевантную информацию в индексированных новостных публикациях, а также получать краткий ответ на вопрос, сформированный языковой моделью на основе найденных документов.

Приложение состоит из:

* Фронтенд-интерфейса, созданного на React
* Бэкенд-API, созданного на FastAPI
* OpenSearch в качестве поисковой системы и системы индексирования
* LLM (через OpenRouter API) для суммаризации найденных документов

## Функции

* Полнотекстовый поиск
* Быстрый поиск с использованием OpenSearch
* Индексирование данных
* RAG-суммаризация — ответ на вопрос пользователя на основе найденных новостей
* REST API бэкенд
* Веб-интерфейс с переключением между режимами "Поиск" и "Спросить"
* Взаимодействие между фронтендом и бэкендом

## Установка

### 1. Клонировать репозиторий

```
git clone https://github.com/ioannya/SearchRIA.git
cd SearchRIA
```

### 2. Поднять OpenSearch (Docker)

Перед запуском backend нужно, чтобы OpenSearch был доступен на `localhost:9200`.

```
docker compose up -d
```

Проверить, что OpenSearch отвечает:

```
curl http://localhost:9200
```

### 3. Настроить переменные окружения

Скопировать `.env.example` в `.env` и заполнить своими значениями:

```
VK_TOKEN=твой_токен_vk_api
LLM_API_KEY=твой_ключ_openrouter
LLM_BASE_URL=https://openrouter.ai/api/v1
LLM_MODEL=tencent/hy3:free
```

Ключ VK API нужен для парсинга (`vk_parser.py`), ключ OpenRouter — для работы RAG-эндпоинта `/ask`. Актуальный список бесплатных моделей можно посмотреть на [openrouter.ai/models](https://openrouter.ai/models).

### 4. Backend

Установить зависимости:

```
pip install -r requirements.txt
```

Запустить сервер:

```
uvicorn main:app --reload
```

Backend запустится на:

```
http://127.0.0.1:8000
```

### 5. Frontend

Перейти в папку frontend:

```
cd frontend
```

Установить зависимости:

```
npm install
```

Запустить:

```
npm run dev
```

Frontend запустится на:

```
http://localhost:5173
```

## API

### `GET /`

Проверка работоспособности сервера.

Возвращает:

```json
{
  "status": "ok"
}
```

### `GET /search?q={query}`

Полнотекстовый поиск по индексированным новостям. Возвращает список найденных документов, отсортированных по релевантности.

Пример:

```
GET /search?q=Россия
```

Ответ:

```json
{
  "query": "Россия",
  "count": 5,
  "results": [
    {
      "post_id": "50183480",
      "text": "...",
      "date": 1721400000,
      "source": "РИА Новости",
      "url": "https://vk.com/wall-15755094_50183480",
      "score": 2.28
    }
  ]
}
```

### `GET /ask?q={query}`

RAG-эндпоинт: находит топ-5 наиболее релевантных документов и формирует краткий ответ на вопрос пользователя с помощью LLM, опираясь только на найденные документы.

Пример:

```
GET /ask?q=Россия
```

Ответ:

```json
{
  "query": "Россия",
  "answer": "В предоставленных документах термин «Россия» упоминается в разных контекстах...",
  "sources": [
    { "url": "https://vk.com/wall-15755094_50183480", "score": 2.28 }
  ]
}
```

## Код сервисов

Хранится в файлах `main.py` (сервис FastAPI, который обрабатывает пользовательские запросы — поиск и RAG-ответ), `summarizer.py` (формирование RAG-ответа через LLM) и `App.jsx` (сервис пользовательского интерфейса React).

## Код скриптов

Хранится в файле `vk_parser.py`:

* Подключается к VK API
* Получает новости
* Сохраняет их в OpenSearch
